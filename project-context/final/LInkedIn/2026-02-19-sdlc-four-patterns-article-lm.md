## Post Copy

Everyone is using AI to write more code faster.

The 2024 DORA report found that for every 25% increase in AI adoption, delivery stability *decreased* by 7.2%.

We are producing more code. We are not delivering more value.

I've been thinking about this for a long time. Yesterday I ran a Lightning Lesson on launching agentic applications to production — and the core of what I presented was a framework I've been developing for how the SDLC is actually evolving with AI coding tools.

Not how vendors want you to think it's evolving. How it's actually evolving, based on the data.

I see four distinct patterns:

**Pattern 1 — Vibe Coding**
Fast, ungoverned, prototype-only. Great for weekend projects. GitClear analyzed 211 million lines of AI-assisted code and found an 8x increase in duplicated code blocks. 45% of AI-generated code fails security tests (Veracode). This is where most teams are operating today.

**Pattern 2 — AI Augmented SDLC**
AI tools assisting individual developers within existing workflows. Productivity gains are real. But the bottleneck has moved — it's no longer individual execution. It's coordination across the lifecycle.

**Pattern 3 — Agentic Architect SDLC**
AI Agent Personas orchestrated by a human architect. The bottleneck shifts from "how fast can we write code" to "how well can we coordinate agents, context, governance, and rollout." This is where production value lives.

**Pattern 4 — Agentic VDLC**
Continuous, autonomous value delivery. Not there yet for most teams — but the architecture decisions you make at Pattern 3 either enable or block this transition.

The most important insight: speed without coordination is the new technical debt. McKinsey's 2025 research found a 15-point performance gap between top and bottom AI-adopting organizations. The differentiator is not tool adoption — it's operating model redesign.

I wrote all of this up in a new long-form article. It's 11 pages, fully sourced, and free.
